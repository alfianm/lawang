§ 01 — RAISON D'ÊTRE

Untuk yang nggak mau
nitip terminal ke cloud orang.

SSH dari HP ribet. Layanan remote-desktop minta akun, billing, dan kadang ngirim trafik kamu ke region antah berantah. Lawang kebalikannya — agent jalan di laptop kamu, izin di tangan kamu.

№ 01

Approval di host, selalu.

Setiap pairing harus dijawab y di terminal kamu sendiri. Trusted device opsional, dan bisa dicabut sewaktu-waktu.

№ 02

Token sekali pakai.

Pairing token base64url 24-byte dengan TTL 15 menit, dihash SHA-256, dan langsung di-consume setelah approved. Tidak ada secret yang nempel di disk.

№ 03

Tunnel opsional.

Auto-deteksi cloudflared kalau ada. Kalau tidak, ya cuma LAN. Pilihan ada di kamu, bukan di vendor.

№ 04

Sandbox keras.

Path traversal ditolak di resolveInside(). File API & git API hidup di dalam root project yang kamu pilih, titik.

№ 05

Audit log lokal.

JSONL append-only di ~/.lawang/audit.log. Pairing, sesi, file write, git push — semua tercatat di mesin kamu.

№ 06

Mobile-first nyata.

xterm.js + shortcut bar (ESC, TAB, ↑/↓, Ctrl-C) untuk jari, bukan mouse. File explorer dan Monaco editor jalan di Safari iOS.

№ 07

Reconnect tanpa kehilangan output.

PTY tetap hidup ~5 menit setelah WebSocket putus. Saat client reconnect, output ring-buffer 256 KB di-replay otomatis. Refresh tab tidak membunuh npm run dev.

№ 08

Local sites proxy.

Expose localhost:3000, :5173, dst. ke browser remote lewat /proxy/<port>/. Bisa allow-list per-port atau mode --proxy open untuk dev.

№ 09

Granular permissions.

Saat approve, host pilih scope: full, read-only, atau terminal-only. Trusted device menyimpan scope yang sama untuk pairing berikutnya.

§ 02 — MEKANIKA

Tiga gerakan,
nol akun.

Dari install sampai shell di HP, semuanya tetap di mesinmu sendiri. Tidak ada langkah yang minta email atau API key.

01

Nyalakan agent.

npm start di root project. Agent bind ke :3999, generate pairing token, render QR di terminal.

02

Scan & request.

Buka kamera HP di QR. Browser load /#/pair?token=…, kirim device fingerprint, masuk antrean approval.

03

Approve, lalu shell.

Jawab y di CLI. Token di-consume, session token diterbitkan, WebSocket ke /ws/terminal terbuka. Kamu sudah di shell.

§ 04 — QUICKSTART

Dari nol ke shell,
sekitar enam baris.

Asumsinya Node 18+ sudah ada. Install, build sekali, jalan, scan, approve. Selesai.

Mode produksi

Build static asset web sekali, lalu tinggal npm start.

~ ❯ cd ~/lawang ~ ❯ npm install ~ ❯ npm run build ~ ❯ npm start Pair URL : …/#/pair?token=… scan, approve, masuk shell.

Mode developer

Dua proses paralel. Vite proxy /api & /ws ke agent.

# terminal 1 ~ ❯ npm run dev:agent # terminal 2 ~ ❯ npm run dev:web open http://localhost:5173
§ 04.1 — FLAGS

Sentuhan terakhir.

CLI lawang start punya flags untuk port, root project, TTL token, idle timeout, dan ukuran QR. Subcommand lawang devices untuk list atau revoke trusted device kapan pun.

~ ❯ lawang start \ --port 4100 \ --root ~/projects/myapp \ --no-tunnel \ --token-ttl 30 ~ ❯ lawang devices --revoke 1a2b3c4d ✓ Revoked: iPhone Safari (1a2b3c4d…)
§ 05 — FAQ

Tanya yang biasa
ditanyakan duluan.

Pertanyaan jujur, jawaban jujur. Termasuk yang tidak bisa dilakukan oleh Lawang.

Apakah laptop saya tetap bisa diakses kalau sleep atau mati?

Tidak. Kalau laptop benar-benar sleep atau shutdown, tidak ada proses yang bisa listen. Itu batas hardware/OS, bukan batas aplikasi.

Yang bisa: jalankan dengan --keep-awake supaya OS tidak sleep selama agent hidup, atau host agent di mesin yang memang dirancang selalu hidup (Raspberry Pi, NAS, VPS).

Bagaimana kalau HP saya mati setelah sukses pairing?

Active sessions otomatis 0, dan pairing token sudah consumed (one-time). Solusinya: jalankan lawang rotate di mesin host. Token baru di-issue tanpa perlu restart agent. Halaman /qr auto-refresh tiap 5 detik, jadi tinggal scan ulang.

Data saya nyangkut di server cloud kalian?

Tidak ada. Repo ini local-first. Audit log, config, dan trusted devices semua di ~/.lawang/ di mesin kamu. Tunnel cloudflared cuma transport, isinya tidak disimpan di mereka. Kalau pakai --no-tunnel, data malah tidak pernah keluar LAN.

Seberapa aman pairing token-nya?

Token random 24-byte base64url, hash SHA-256, TTL 15 menit, one-time. Kalau approved → di-consume; kalau gagal → expired. Pairing endpoint rate-limited 20/menit per IP. WebSocket terminal cek Origin header dan tolak frame > 64 KB.

Run lawang verify kapan saja untuk smoke-test hard rules ini.

Saya bisa pakai vim atau htop di tab Chat?

Tidak. Tab Chat pakai one-shot exec, tidak ada PTY interaktif. UI mendeteksi command interaktif dan menyarankan pindah ke tab Terminal yang memang pakai node-pty + xterm.js.

Bisa di-host di Raspberry Pi / VPS?

Bisa. Code identik. Install Node 18+, clone repo, npm install, npm run build, lalu npm start. Akses tetap via cloudflared tunnel, tidak butuh konfigurasi tambahan. Power features otomatis adapt ke Linux (systemctl).

Windows kapan?

Code mendukung Windows secara teori (node-pty, deteksi platform). Tapi belum ada smoke test penuh. Power actions di Windows belum ditambahkan karena UAC bikin banyak edge case. Kontribusi welcome.

Saya kontribusi bagaimana?

Repo MIT, fork & PR. Mulai dari issue kecil dulu — Windows compat test, FAQ tambahan, atau dokumentasi. Untuk fitur besar (remote desktop, multi-host), buka diskusi dulu sesuai roadmap di prd_remote_access_app.md.
