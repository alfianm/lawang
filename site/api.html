Endpoint, terbuka apa adanya.

Semua endpoint di-host oleh agent yang sama. Untuk curl, base URL biasanya http://localhost:3999 atau URL cloudflared yang dicetak CLI saat start.

Auth. Endpoint di luar zona public butuh header Authorization: Bearer <sessionToken>. Session token keluar dari respons POST /api/pair/request setelah host approve.

§ APublic

GET/healthno auth

Health probe ringan. Aman dipakai untuk uptime check.

curl -s http://localhost:3999/health
{ "status": "ok", "machineName": "your-machine", "version": "0.1.0" }
GET/api/infono auth

Info publik tentang agent. Dipakai oleh halaman Pair untuk render machine name.

curl -s http://localhost:3999/api/info
{
  "machineName": "your-machine",
  "version": "0.1.0",
  "tunnelUrl": "https://soft-fern.trycloudflare.com",
  "pairUrl": "https://soft-fern.trycloudflare.com/#/pair?token=k3X…"
}

pairUrl selalu berisi token pairing terbaru. Halaman /qr mempolling endpoint ini setiap 5 detik untuk auto-refresh QR setelah lawang rotate.

GET/qr · /qr.svgno auth

Halaman HTML full-screen QR (untuk dibuka di laptop) dan SVG mentah QR pairing yang aktif. URL pair di-rotate setiap kali agent di-restart.

open http://localhost:3999/qr            # mac
curl http://localhost:3999/qr.svg > qr.svg
POST/api/pair/requestno auth · rate-limited 20 / menit

Kirim pairing token + identitas device. Server akan menahan koneksi sampai host menjawab di CLI (atau token kadaluarsa). Trusted device lewat fingerprint cocok → auto-approve tanpa prompt.

pairingToken*Raw token dari URL ?token=…, 8–256 char.
deviceNameString 1–80 char. Default: deteksi user-agent.
deviceTypemobile · desktop · unknown.
deviceFingerprintRandom ID per device (lihat localStorage client). Wajib kalau ingin trust.
curl -s -X POST http://localhost:3999/api/pair/request \
  -H 'content-type: application/json' \
  -d '{
    "pairingToken": "k3X…",
    "deviceName": "iPhone Safari",
    "deviceType": "mobile",
    "deviceFingerprint": "9d7c-…"
  }'
{
  "status": "approved",
  "sessionToken": "S_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
  "machineName": "your-machine",
  "permissions": ["terminal","file:read","file:write","git:read","git:write"],
  "trusted": true
}

Status lain: rejected, rate_limited (HTTP 429), invalid_request (HTTP 400).

§ BSession

GET/api/sessionbearer required

Info sesi yang sedang aktif: machine, root path, dan permission yang dipegang token.

curl -s http://localhost:3999/api/session \
  -H "Authorization: Bearer $TOKEN"
{
  "sessionId": "ad7e…",
  "machineName": "your-machine",
  "rootPath": "~/lawang",
  "permissions": ["terminal","file:read","file:write","git:read","git:write"],
  "deviceName": "iPhone Safari",
  "createdAt": "2026-05-23T15:00:11.231Z"
}
GET/api/envbearer required

Auto-detect environment host: machine info, runtime Node, shell aktif, dan ringkasan project (package manager, lockfile, git repo, workspaces). Berguna untuk render badge di UI atau menyesuaikan command suggestion.

  -H "Authorization: Bearer $TOKEN"
{
  "machine":  { "hostname": "your-machine", "platform": "darwin", "arch": "arm64", "cpus": 8, "release": "24.6.0" },
  "runtime":  { "node": "22.20.0", "npm_user_agent": null },
  "shell":    { "path": "/bin/zsh", "name": "zsh" },
  "project":  {
    "rootPath": "~/lawang",
    "name": "lawang-monorepo",
    "version": "0.1.0",
    "packageManager": "npm",
    "packageManagerLockfile": "package-lock.json",
    "isGitRepo": true,
    "monorepo": true,
    "workspaces": ["packages/*"]
  }
}

§ CFiles

Semua path bersifat relatif terhadap rootPath sesi. Path traversal (..) ditolak oleh resolveInside(). Rate limit: 240/menit untuk read, 60/menit untuk write.

GET/api/files?path=<rel>bearer required

List entri direktori. Sort: dir dulu, lalu alfabet.

curl -s "http://localhost:3999/api/files?path=src" \
  -H "Authorization: Bearer $TOKEN"
{
  "path": "src",
  "rootName": "remoteapp",
  "entries": [
    { "name": "lib", "path": "src/lib", "type": "dir",  "size": 0,    "modifiedAt": "…" },
    { "name": "cli.ts","path": "src/cli.ts","type":"file","size": 7340, "modifiedAt": "…" }
  ]
}
GET/api/file?path=<rel>bearer required

Baca file. Maks 1 MB. Binary di-encode base64; teks dikirim apa adanya.

curl -s "http://localhost:3999/api/file?path=README.md" \
  -H "Authorization: Bearer $TOKEN"
{
  "path": "README.md",
  "size": 1023,
  "modifiedAt": "…",
  "encoding": "utf8",
  "content": "# Lawang\n…",
  "isBinary": false,
  "mime": "text/markdown"
}
GET/api/file/download?path=<rel>&token=<sessionToken>token in query

Stream file biner. Token diteruskan via query agar bisa dipakai langsung di <a download> tanpa setup header. Gunakan dengan hati-hati di environment shared.

curl -L -o snapshot.png \
  "http://localhost:3999/api/file/download?path=docs/snapshot.png&token=$TOKEN"
PUT/api/filebearer required

Tulis ulang file. Body JSON dengan encoding utf8 (default) atau base64. Body content dibatasi ~8 MB base64.

curl -s -X PUT http://localhost:3999/api/file \
  -H "Authorization: Bearer $TOKEN" \
  -H 'content-type: application/json' \
  -d '{
    "path": "notes.md",
    "content": "# Hello\nDitulis dari HP.",
    "encoding": "utf8"
  }'
{ "status": "ok", "path": "notes.md", "size": 28, "modifiedAt": "…" }
POST/api/file/upload?path=<rel>bearer required

Upload streaming. Kirim raw body (application/octet-stream); body limit Fastify 16 MB. Untuk file besar, ini lebih efisien daripada base64.

curl -s -X POST \
  "http://localhost:3999/api/file/upload?path=uploads/photo.jpg" \
  -H "Authorization: Bearer $TOKEN" \
  -H 'content-type: application/octet-stream' \
  --data-binary @photo.jpg
{ "status": "ok", "path": "uploads/photo.jpg", "size": 482311 }
POST/api/dirbearer required

Buat direktori (recursive, mkdir -p).

curl -s -X POST http://localhost:3999/api/dir \
  -H "Authorization: Bearer $TOKEN" \
  -H 'content-type: application/json' \
  -d '{ "path": "drafts/2026" }'
{ "status": "ok", "path": "drafts/2026" }
POST/api/file/renamebearer required

Pindah / rename. Target tidak boleh sudah ada (HTTP 409 conflict).

curl -s -X POST http://localhost:3999/api/file/rename \
  -H "Authorization: Bearer $TOKEN" \
  -H 'content-type: application/json' \
  -d '{ "from": "notes.md", "to": "drafts/notes.md" }'
{ "status": "ok", "from": "notes.md", "to": "drafts/notes.md" }
DEL/api/file?path=<rel>bearer required

Hapus file atau folder (recursive). Root project tidak bisa dihapus (HTTP 403).

curl -s -X DELETE \
  "http://localhost:3999/api/file?path=drafts/old.md" \
  -H "Authorization: Bearer $TOKEN"
{ "status": "ok" }

§ EChat / Exec

Endpoint one-shot exec untuk tab Chat. Server menjalankan command di shell login ($SHELL -lc di Unix, cmd /d /s /c di Windows) dengan cwd di-resolve lewat sandbox yang sama, env TERM=dumb + NO_COLOR=1 agar output bersih, dan output combined dibatasi 256 KB. Default timeout 15 detik (max 60).

POST/api/execbearer required · perm: terminal
command*String 1–4000 char. Diteruskan ke shell sebagai -lc <command>.
cwdRelatif terhadap root project. Default ".".
timeoutMs1000–60000. Default 15000.
curl -s -X POST http://localhost:3999/api/exec \
  -H "Authorization: Bearer $TOKEN" \
  -H 'content-type: application/json' \
  -d '{ "command": "git status -s", "cwd": "." }'
{
  "cwd": ".",
  "command": "git status -s",
  "stdout": " M README.md\n?? site/\n",
  "stderr": "",
  "exitCode": 0,
  "signal": null,
  "durationMs": 84,
  "truncated": false,
  "timedOut": false
}

Catatan: command interaktif (vim, htop, ssh) tidak akan jalan di sini—pakai tab Terminal untuk itu.

§ EWebSocket

WS/ws/terminal?token=<sessionToken>token in query

Pipe ke PTY shell host. Pesan dikirim sebagai JSON { event, payload }. Auto-close dengan code 4401 kalau token tidak valid.

Server → client

  • session:connected — payload: { machineName, sessionId, cwd, shell }.
  • terminal:output — payload: { data: string } (chunk PTY stdout).
  • terminal:exit — payload: { exitCode, signal }.
  • error — payload: { code, message }.

Client → server

  • terminal:input — payload: { data: string }.
  • terminal:resize — payload: { cols, rows }.
  • session:disconnect — minta server menutup koneksi.

Smoke test (websocat)

websocat "ws://localhost:3999/ws/terminal?token=$TOKEN"
# kirim:
{"event":"terminal:resize","payload":{"cols":120,"rows":32}}
{"event":"terminal:input","payload":{"data":"echo hello\n"}}

§ FHost power

Trigger sleep / shutdown pada mesin host. Server menjalankan command sebagai user yang menjalankan agent — di macOS pakai pmset sleepnow atau osascript graceful shutdown (tanpa sudo karena user adalah console user aktif), di Linux pakai systemctl suspend|poweroff (biasanya tanpa password berkat polkit pada session aktif). Windows belum didukung.

GET/api/system/powerbearer required · perm: terminal

Kapabilitas sleep/shutdown untuk host saat ini.

curl -s http://localhost:3999/api/system/power \
  -H "Authorization: Bearer $TOKEN"
{
  "sleep":    { "supported": true,  "provider": "pmset" },
  "shutdown": { "supported": true,  "provider": "osascript", "needsAuth": false },
  "reboot":   { "supported": true,  "provider": "osascript" },
  "lock":     { "supported": true,  "provider": "pmset displaysleepnow" }
}
POST/api/system/powerbearer required · perm: terminal · confirm wajib
action*sleep, shutdown, reboot, atau lock. lock default delay 0; lainnya 5 detik.
confirm*Wajib literal true. Tanpa ini server balas HTTP 400 confirmation_required.
delaySeconds0–120, default 5. HTTP response balik dulu, baru host shutdown setelah delay.
curl -s -X POST http://localhost:3999/api/system/power \
  -H "Authorization: Bearer $TOKEN" \
  -H 'content-type: application/json' \
  -d '{ "action": "sleep", "confirm": true }'
{
  "status": "queued",
  "action": "sleep",
  "delaySeconds": 5,
  "willHappenAt": "2026-05-23T16:42:11.000Z"
}

UI di Session header menyediakan tombol host dengan modal konfirmasi double-tap.

GET/api/system/batterybearer required · perm: file:read

Info baterai host. Polling tiap menit di UI untuk render badge baterai di header. macOS pakai pmset -g batt, Linux pakai sysfs /sys/class/power_supply.

curl -s http://localhost:3999/api/system/battery \
  -H "Authorization: Bearer $TOKEN"
{
  "supported": true,
  "hasBattery": true,
  "percent": 62,
  "charging": false,
  "acConnected": false,
  "state": "discharging",
  "timeRemainingMin": 397,
  "source": "pmset"
}

Mesin tanpa baterai (desktop on AC) → hasBattery: false, acConnected: true. UI menampilkan badge "AC" tanpa percent.

§ GError shape

Server menjawab dengan HTTP status idiomatik plus body JSON kecil. Kode yang sering muncul:

HTTPstatuskapan
400invalid_requestBody tidak lolos zod / parameter hilang.
401unauthorizedToken tidak ada / expired.
403forbiddenPath di luar root, atau permission tidak cukup.
404not_foundFile / route tidak ada.
409conflict / not_a_repoTarget rename sudah ada, atau bukan repo git.
413too_largeFile > 1 MB untuk preview, atau body > 16 MB.
429rate_limitedPairing > 20/menit, file read > 240/menit, file write > 60/menit.
