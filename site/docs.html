01Apa itu Lawang

Lawang adalah local-first remote terminal untuk mesin sendiri. Host menjalankan satu perintah CLI, lalu device lain (HP, tablet, laptop) bisa scan QR code untuk mendapat akses terminal di browser. Setiap pairing harus di-approve secara eksplisit oleh host.

Cocok untuk:

  • Membuka terminal Mac/Linux dari HP saat lagi di luar.
  • Cek file dan git status dari device kedua.
  • Coding ringan dengan Monaco editor langsung di browser.

Fitur MVP yang sudah jalan

  • CLI agent yang menyalakan HTTP + WebSocket server lokal.
  • Pairing one-time via QR + approval di CLI host.
  • Terminal berbasis xterm.js dengan shortcut bar untuk mobile.
  • File explorer + Monaco editor (read/write, upload, rename, delete, mkdir).
  • Git panel (status, diff, log, stage, commit, pull, push).
  • Tab Chat: ketik command satu per satu, tiap perintah jadi bubble dengan output yang dirapikan (JSON, diff, exit code).
  • Tunnel opsional via cloudflared (auto-detect).
  • Trusted devices (auto-approve) dan audit log lokal.
  • Control socket lokal untuk operasi admin di luar HTTP.

02Tech stack

Repo & runtime

  • npm workspaces monorepo (packages/*).
  • Node.js >= 18.17.
  • TypeScript 5.5.4 di kedua package.

Agent (packages/agent)

  • fastify@4 — HTTP server.
  • @fastify/static — serve build web.
  • @fastify/websocket + ws@8 — WS terminal.
  • node-pty@1 — PTY shell (bash/zsh/powershell).
  • commander@12 — CLI parsing.
  • qrcode + qrcode-terminal — render QR.
  • simple-git — operasi git.
  • zod — validasi body request.
  • tsx dev / tsc build.

Web (packages/web)

  • React 18.3 + Vite 5.4.
  • Tailwind CSS 3.4 dengan tema dark custom.
  • @xterm/xterm + addon fit & web-links.
  • @monaco-editor/react + monaco-editor.
  • lucide-react untuk ikon.
  • Hash router custom di packages/web/src/lib/router.ts.

04Arsitektur & alur

Komponen runtime

  • Fastify mengikat 0.0.0.0:<port> (default 3999).
  • Web di-serve sebagai static asset dari packages/agent/public.
  • Tunnel cloudflared dipakai sebagai public origin kalau binary tersedia.
  • Audit & config di ~/.lawang dengan permission 0o600.

Alur pairing

  1. Host jalan lawang start.
  2. CLI generate pairing token (24-byte base64url, TTL 15 menit) dan render QR + URL pair (<base>/#/pair?token=…).
  3. Device buka URL → Pair.tsx POST /api/pair/request dengan token, device name, type, dan fingerprint (random per device, di localStorage).
  4. Server validasi token (PairingManager.validateRawToken) dan cek trusted device. Cocok → auto-approve. Tidak → masuk antrean prompt host.
  5. Approved → token consume (one-time), session token (32-byte random, hash di memori) terbit dengan permission terminal, file:read, file:write, git:read, git:write.
  6. Client simpan session token di sessionStorage, navigate ke /#/terminal.
  7. Client buka WS /ws/terminal?token=…. Server validasi, spawn PTY, stream terminal:output / terminal:input.

Sesi & idle timeout

  • SessionStore in-memory, idle timeout default 30 menit, sweeper tiap 60 detik.
  • Setiap REST/WS valid memanggil touch(sessionId).
  • Sesi bisa direvoke via control socket (sessions, revoke, revoke-all).
  • Saat SIGINT/SIGTERM, semua sesi di-revoke dan tunnel dihentikan.

Sandbox file & git

  • Semua endpoint file/git lewat resolveInside(root, relative).
  • File preview maks 1 MB; upload streaming dengan body limit 16 MB.
  • Rate limit per-IP: pairing 20/menit, file read 240/menit, file write 60/menit.

Audit log

File ~/.lawang/audit.log append-only JSONL. Event antara lain agent_started, pairing_requested, pairing_approved, session_started, session_revoked, file_written, git_commit, git_push. Lihat AuditEventType di packages/agent/src/lib/audit.ts.

05Endpoint ringkas

Semua endpoint kecuali yang ditandai publik membutuhkan header Authorization: Bearer <sessionToken>.

Publik

MethodPathKeterangan
GET/healthHealth probe
GET/api/infoMachine name, version, tunnel URL
GET/qr · /qr.svgHalaman & SVG QR pairing
POST/api/pair/requestRequest pairing (rate limited)

Environment

GET /api/env — info host: machine, runtime Node, shell, plus ringkasan project (name, version, package manager, lockfile, git repo, workspaces).

Session history

GET /api/sessions/history?limit=<n> — sesi yang sudah lewat dari audit log. Cocok untuk render daftar sesi di UI tanpa perlu menunggu agent restart.

File API

MethodPathKeterangan

GET/api/file?path=<rel>Baca file (auto text/binary)
GET/api/file/downloadDownload stream
PUT/api/fileTulis ulang file
POST/api/file/uploadUpload streaming octet-stream
POST/api/dirmkdir -p
POST/api/file/renamePindah file/folder
DELETE/api/fileHapus file/folder

Git API

MethodPath
GET/api/git/status · /api/git/diff · /api/git/log
POST/api/git/stage · /api/git/unstage
POST/api/git/commit · /api/git/pull · /api/git/push

Chat (one-shot exec)

POST /api/exec — jalankan command sekali (bukan PTY). Body { command, cwd, timeoutMs }. Output combined dibatasi 256 KB, default timeout 15 detik. Cocok untuk tab Chat di UI; bukan pengganti tab Terminal untuk command interaktif.

Host power

GET /api/system/power — kapabilitas. POST /api/system/power dengan { action, confirm: true, delaySeconds } men-trigger sleep / shutdown / reboot / lock di host. macOS pakai pmset & osascript, Linux pakai systemctl + loginctl lock-session. Tombol host di header sesi membuka menu dengan modal konfirmasi double-tap (kecuali lock yang langsung).

Battery

GET /api/system/battery — info baterai host: percent, charging, AC connected, state, sisa waktu. Polling tiap menit di UI untuk render badge di header sesi.

WebSocket

GET /ws/terminal?token=<sessionToken> — pesan JSON { event, payload } dengan event session:connected, terminal:output, terminal:input, terminal:resize, terminal:exit, session:disconnect.

06Cara menjalankan

Prasyarat

  • Node.js >= 18.17.
  • npm >= 9.
  • Toolchain native untuk node-pty — compiler C/C++ + node-gyp. Detail kenapa wajib ada ada di Kenapa node-pty butuh compiler native. Ringkasnya:
    • macOS — xcode-select --install (memasang clang, header sistem, make).
    • Linux — python3, make, g++ (Debian/Ubuntu: apt install build-essential python3).
    • Windows — Visual Studio Build Tools 2022, workload Desktop development with C++, plus python3.
  • Opsional: cloudflared di PATH untuk tunnel publik.

Kenapa node-pty butuh compiler native

node-pty membungkus syscall pseudo-terminal milik OS — forkpty di Unix, ConPTY/winpty di Windows. Karena fitur ini berasal dari kernel dan tidak bisa diimplementasi pakai JavaScript murni, package-nya berbentuk native addon: source C/C++ yang harus dikompilasi jadi binary .node saat npm install.

Tanpa native PTY, terminal di browser tidak bisa benar-benar menjalankan bash/zsh dengan ANSI color, resize via SIGWINCH, job control (Ctrl+C/Ctrl+Z), prompt interaktif (sudo, git push), maupun TUI seperti vim/htop. Semua itu butuh PTY asli, bukan sekadar child_process.

Saat install, node-gyp dipanggil dan butuh tiga hal:

  • Compiler C/C++ — meng-compile source node-pty ke binary asli platform.
  • make — build runner yang dipakai node-gyp.
  • python3 — orchestrator node-gyp (bukan untuk runtime).

Hasilnya tersimpan di node_modules/node-pty/build/Release/pty.node dan di-cache. Toolchain ini hanya disentuh sekali, kecuali kamu pindah versi Node mayor, upgrade OS yang mengubah ABI, atau pindah arsitektur (Intel → Apple Silicon). Saat itu, hapus node_modules lalu npm install ulang.

Indikator gagal install yang sering ketemu:

  • gyp ERR! find Python → install python3 lalu retry.
  • xcrun: error: invalid active developer path → jalankan xcode-select --install.
  • error: Microsoft Visual C++ 14.0 or greater is required → install Visual Studio Build Tools dengan workload C++.

Kalau ingin menghindari toolchain di server produksi yang minim, build dulu di mesin developer, lalu copy node_modules atau pakai Docker multi-stage build.

Install dependency

cd ~/lawang
npm install

Build production

npm run build
# 1. npm run build -w lawang-web   → vite ke packages/web/dist
# 2. npm run build -w lawang → tsc + scripts/copy-web.js

Start agent

npm start
# setara: node packages/agent/dist/cli.js start

CLI akan mencetak Local URL, LAN URL, Tunnel URL (kalau cloudflared ada), Pair URL + token, dan QR di terminal. Buka <localUrl>/qr untuk QR full screen di laptop. Stop dengan Ctrl+C.

Mode development

# Terminal 1 — Vite dev (proxy /api & /ws ke localhost:3999)
npm run dev:web

# Terminal 2 — Agent dengan tsx watch
npm run dev:agent

Opsi CLI

  • -p, --port <port> — default 3999.
  • -r, --root <path> — default process.cwd().
  • --no-tunnel — matikan auto-start cloudflared.
  • --token-ttl <menit> — default 15.
  • --idle-timeout <menit> — default 30.
  • --qr-size <small|large|off> — default large.
  • --keep-awake — cegah host sleep selama agent jalan (caffeinate / systemd-inhibit).

Subcommand devices

lawang devices                 # list trusted devices
lawang devices --revoke 1a2b3c # revoke by id / prefix
lawang devices --revoke-all    # revoke semua
lawang devices --json          # output JSON

Subcommand sessions / ping

Butuh agent yang sedang jalan (lewat control socket).

lawang sessions                # list sesi aktif
lawang sessions --revoke ad7e  # revoke sesi by prefix
lawang sessions --revoke-all   # revoke semua sesi aktif
lawang ping                    # cek agent up

Subcommand history

Offline. Baca ~/.lawang/audit.log dan rekonstruksi sesi yang lewat.

lawang history                 # 25 sesi terakhir
lawang history --limit 100     # batasi jumlah
lawang history --json          # output JSON

Subcommand rotate

Generate pairing token baru di agent yang sedang jalan, tanpa restart. Halaman /qr auto-refresh QR-nya tiap 5 detik, jadi tinggal scan ulang dari HP. Sesi aktif dan trusted device tidak terganggu.

lawang rotate                  # token baru, banner di-print ulang
lawang rotate --json           # output JSON (pairUrl, token, expiresAt)

Subcommand verify

Smoke-test hard rules dari PRD § 17 / 18.3 terhadap agent yang sedang jalan: control ping, /health, no unauthenticated access, path traversal guard, push confirmation, dan WS Origin check. Exit 0 kalau semua lulus, 1 kalau ada yang fail.

lawang verify                  # 6 checks
lawang verify --base http://localhost:4100
lawang verify --json           # CI-friendly output

Lokasi data runtime

  • ~/.lawang/config.json — machine id, trusted devices, settings.
  • ~/.lawang/audit.log — audit JSONL.
  • ~/.lawang/agent.sock (Unix) atau agent.port (Windows).

07Akses dari mana saja (Cloudflared)

Section ini buat user yang baru install Lawang dan mau akses dari HP saat di luar Wi-Fi rumah. Kalau cukup di Wi-Fi yang sama, lewati saja.

Tiga skenario, tiga jawaban

  • HP & laptop di Wi-Fi sama → tidak perlu apa-apa. lawang start --no-tunnel, scan QR.
  • HP di 4G / kafe lain → install cloudflared di laptop, lalu lawang start. URL random publik otomatis muncul di banner.
  • Mau URL tetap (mis. lawang.namadomain.com) → butuh akun Cloudflare + domain. Lihat sub-section "URL stabil" di bawah.

Install cloudflared

Cloudflared dipasang di laptop user yang menjalankan Lawang, bukan di server developer. HP user tidak perlu cloudflared sama sekali.

macOS

brew install cloudflared

Linux Debian / Ubuntu

curl -L --output cloudflared.deb \
  https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
sudo dpkg -i cloudflared.deb

Linux Fedora / RHEL

curl -L --output cloudflared.rpm \
  https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-x86_64.rpm
sudo rpm -i cloudflared.rpm

Linux generic

curl -L --output cloudflared \
  https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
chmod +x cloudflared
sudo mv cloudflared /usr/local/bin/

Windows

winget install --id Cloudflare.cloudflared
# atau:
scoop install cloudflared

Verifikasi: cloudflared --version dan which cloudflared (atau where di Windows).

Tutorial end-to-end (skenario paling sering dipakai)

Laptop di rumah, HP di luar (4G atau Wi-Fi lain).

  1. Pasang cloudflared (lihat di atas).
  2. lawang start --keep-awake — flag keep-awake mencegah laptop sleep selama agent jalan.
  3. Banner CLI cetak Tunnel URL: https://soft-fern.trycloudflare.com dan QR.
  4. Scan QR dari HP. Boleh di Wi-Fi manapun atau 4G.
  5. Approve di laptop dengan y. Optional trust device untuk auto-approve berikutnya.
  6. Sesi terbuka. Tab Terminal default, ada Files, Git, Chat juga.
  7. HP mati / putus / scan ulang? Jalankan lawang rotate di laptop. Token baru, tanpa restart agent.

Yang harus dipahami soal cloudflared

  • Cloudflared dipasang di laptop user, bukan di server developer Lawang. Setiap user punya cloudflared masing-masing.
  • URL random tiap restart agent. Untuk URL tetap, butuh akun Cloudflare.
  • Data tidak nyangkut di server siapapun. Cloudflare cuma transport pipe terenkripsi. Audit log, file, command tetap di laptop user.
  • Cloudflare lihat metadata (IP yang konek, volume traffic), bukan konten.
  • Bandwidth dipakai dari koneksi internet user, bukan dari developer Lawang.

URL stabil dengan named tunnel (advanced)

Hanya kalau user mau URL tidak random. Skip kalau pakai Lawang santai.

Prasyarat: akun Cloudflare gratis (cloudflare.com), domain dengan nameserver pointing ke Cloudflare.

# 1. Login (browser terbuka)
cloudflared tunnel login

# 2. Bikin tunnel
cloudflared tunnel create lawang
# catat UUID dari output

# 3. Buat config
cat > ~/.cloudflared/config.yml <<EOF
tunnel: <UUID>
credentials-file: ~/.cloudflared/<UUID>.json

ingress:
  - hostname: lawang.namadomain.com
    service: http://localhost:3999
  - service: http_status:404
EOF

# 4. Routing DNS otomatis
cloudflared tunnel route dns lawang lawang.namadomain.com

# 5. Jalankan tunnel di terminal terpisah
cloudflared tunnel run lawang

# 6. Jalankan Lawang dengan --no-tunnel (tunnel sudah dipegang manual)
lawang start --no-tunnel

Troubleshooting

  • cloudflared not found. Falling back to local-only access. — binary tidak ada di PATH. Cek which cloudflared. Restart terminal kalau baru install.
  • Timed out waiting for tunnel URL — Cloudflare lambat respons. Coba lagi atau pakai --no-tunnel sementara.
  • HP error 502 / 521 — Cloudflare aktif tapi laptop tidak respond. Cek lawang start masih jalan, cek firewall laptop tidak blokir port 3999.
  • HP error CERT_INVALID — pakai URL https://*.trycloudflare.com, bukan IP langsung.

08Konfigurasi tambahan

Cloudflared tunnel

  • Pasang cloudflared, agent auto-spawn cloudflared tunnel --url http://localhost:<port>.
  • URL di-parse dari log (https://<sub>.trycloudflare.com).
  • Kalau gagal/timeout 20 detik → fallback LAN/local.
  • Matikan dengan --no-tunnel.

Trusted devices

  • Saat host jawab y, ada pertanyaan lanjutan untuk trust device.
  • Fingerprint di-hash SHA-256, disimpan di config.
  • Pairing berikutnya dari device sama → auto-approve.
  • Revoke: lawang devices --revoke <id>.

Permissions

Permission default per sesi: terminal, file:read, file:write, git:read, git:write. Endpoint memvalidasi permission yang dibutuhkan.

09Keamanan

  • Pairing token one-time — di-consume setelah approved.
  • Hash semua secret — pairing token, session token, fingerprint = SHA-256 (tokens.ts).
  • Path traversal guard — resolveInside di sandbox.ts.
  • Body limit — Fastify 16 MB; preview 1 MB; PUT /api/file ~8 MB base64.
  • Bind address — 0.0.0.0. Untuk akses publik, andalkan tunnel.
  • Audit dulu sebelum revoke — cek ~/.lawang/audit.log.

10Troubleshooting

  • npm install gagal di node-pty — install toolchain native (Xcode CLT, build-essential, atau Visual Studio Build Tools). Latar belakang lengkap kenapa toolchain ini wajib ada di Kenapa node-pty butuh compiler native.
  • cloudflared not found — brew install cloudflared atau setara di OS lain.
  • QR tidak terbaca — pakai --qr-size large atau buka /qr di laptop.
  • Token expired — restart agent atau perpanjang --token-ttl.
  • Approval prompt tidak muncul — pastikan terminal punya stdin (jangan service tanpa TTY); pakai trusted device atau wrap tmux.
  • WebSocket auto-close 4401 — session token salah/expired, login ulang dari pair URL.

11Menjalankan Lawang sebagai service

Default lawang start berjalan di foreground. Untuk hosting di VPS / Pi / Mac yang harus selalu reachable, pakai service manager.

Pilih sesuai konteks

  • Test cepat di mesin sendiri → tmux / nohup.
  • Linux VPS atau Pi → lawang install-service (systemd).
  • Mac sebagai server → lawang install-service (launchd).
  • Sudah pakai PM2 → kombinasikan.
  • Container infra → Docker dengan --restart unless-stopped.

install-service (cara paling rapi)

Generate file systemd user unit (Linux) atau launchd plist (macOS), lalu register:

# Linux
lawang install-service --root ~/projects/myapp --no-tunnel --register

# macOS
lawang install-service --root ~/projects/myapp --register

# Cek status
lawang service-status

# Hapus
lawang uninstall-service

# Lihat isi unit tanpa menulis ke disk
lawang install-service --print

Linux: untuk supaya service tetap jalan tanpa user login, tambahkan loginctl enable-linger $USER.

tmux / nohup

tmux new -s lawang
lawang start --keep-awake
# detach: Ctrl+B, D    reattach: tmux attach -t lawang

nohup lawang start > ~/lawang.log 2>&1 &
# stop: pkill -f "lawang start"

PM2

npm i -g pm2
pm2 start "lawang start --no-tunnel" --name lawang
pm2 save
pm2 startup

Docker

docker run -d \
  --name lawang \
  --restart unless-stopped \
  -p 3999:3999 \
  -v $HOME:/work \
  -w /work \
  yourname/lawang

Catatan keamanan

  • Service jalan sebagai user yang menjalankan install-service. Tidak pakai sudo.
  • Audit log tetap di ~/.lawang/audit.log.
  • systemd unit punya hardening minimal: NoNewPrivileges, ProtectSystem=full, PrivateTmp.
  • launchd plist set PATH default supaya cloudflared / git ditemukan saat agent di-spawn.
